# Expense Tracker (家計簿アプリ) - Next.js 学習ガイド

このドキュメントは、新しく追加された `Expense Tracker` を通じて、Next.js (App Router) と React の主要な機能を学習するためのガイドです。

## 1. アプリの構成とデータフロー

このアプリは、以下の順序でデータが流れ、画面が構築されます。コードを読む際も、この順番で追っていくと理解しやすいです。

### 読むべきファイルの順番

1. **`app/expense-tracker/types.ts`**
   - **役割**: データの形（型）と、初期表示用のダミーデータを定義しています。
   - **学ぶこと**: TypeScriptの `interface` や `type` を使った型定義の基本。

2. **`app/expense-tracker/page.tsx`** (Server Component)
   - **役割**: アプリの「入り口」となるページです。ここでサーバー側で初期データを用意します。
   - **学ぶこと**: App Routerにおける **Server Component** の基本。データの取得（今回はモックですが、本来はDB等）と、子コンポーネント（Client Component）へのデータの受け渡し。

3. **`app/expense-tracker/ExpenseDashboard.tsx`** (Client Component)
   - **役割**: アプリの「司令塔」です。サーバーから受け取ったデータを元に、UIを組み立てます。
   - **学ぶこと**: ファイル先頭の `'use client';` 宣言。サーバーから渡された **props** の受け取り。後述のカスタムフックの呼び出し。

4. **`app/expense-tracker/useExpenseManager.ts`** (カスタムフック)
   - **役割**: データの保存、追加、削除、計算などの「裏側のロジック」を全てここにまとめています。
   - **学ぶこと**:
     - **`useState`**: 現在の支出データを保持・更新する仕組み。
     - **`useEffect`**: 画面表示時（マウント時）にローカルストレージからデータを読み込む処理や、データが変わった時に自動保存する「副作用」の仕組み。
     - **`useMemo`**: 合計金額などを効率よく計算（キャッシュ）する仕組み。
     - **カスタムフックの概念**: これらをひとまとめにして、コンポーネントをスッキリさせる技術。

5. **`app/expense-tracker/components/` 以下のファイル群** (UIコンポーネント)
   - **役割**: 画面の各部品（フォーム、リスト、1行分のデータ）の見た目を定義します。
   - **学ぶこと**: 親（`ExpenseDashboard`）から **props** としてデータや関数（追加・削除の処理）を受け取り、それを元に画面を描画したり、ボタンクリックで親の関数を実行する仕組み。

## 2. 各機能の詳細解説

### サーバーコンポーネント (Server Components)
`page.tsx` はデフォルトでサーバーコンポーネントとして動きます。
ブラウザ（クライアント）にJavaScriptを送る前に、サーバー側でHTMLを生成するため、初期表示が速く、SEOに有利です。ここで初期データを取得し、それをClient Componentに渡すのがApp Routerの基本設計です。

### クライアントコンポーネント (Client Components)
`ExpenseDashboard.tsx` や `ExpenseForm.tsx` には先頭に `'use client';` と書いています。
ユーザーがボタンをクリックしたり、文字を入力したりする「イベント（動き）」を扱う場合や、ブラウザの機能（ローカルストレージなど）を使う場合は、Client Componentにする必要があります。

### props (プロップス)
コンポーネント間でデータを受け渡すための仕組みです。
- `page.tsx` → `ExpenseDashboard` には初期データ（配列）を渡しています。
- `ExpenseDashboard` → `ExpenseForm` には、新しいデータを追加するための「関数（`addExpense`）」を渡しています。
- これにより、親がデータとロジックを持ち、子はただそれを表示して操作を親に伝えるだけ、という役割分担ができます。

### useEffect
「コンポーネントが表示された時」や「特定のデータが変わった時」に、Reactの描画とは別の処理（副作用）を実行するためのフックです。
`useExpenseManager.ts` では、2つの `useEffect` を使っています。
1. 画面が表示された最初の一回だけ実行し、過去のデータをローカルストレージから読み込む。
2. 支出データ（`expenses`）が更新されるたびに実行し、最新のデータをローカルストレージに保存する。

### カスタムフック (`useExpenseManager`)
`useState` や `useEffect` などのReactの標準フックを組み合わせて、独自の機能を持たせた関数です。
もしこれを `ExpenseDashboard.tsx` の中に全て書くとコードが非常に長くなり読みづらくなりますが、外に切り出すことで「見た目（UI）」と「ロジック（仕組み）」を綺麗に分離することができます。

## 3. まとめ

このアプリのコードには、各所に詳細なコメントが書かれています。
まずは上記の「読むべきファイルの順番」に沿ってコードとコメントを読み進め、データがどのように親から子へ渡され、そしてユーザーの操作がどのように親に伝わってデータを更新するのか（単方向データフロー）を意識して学習してみてください。
