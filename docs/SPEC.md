# プロジェクト仕様書

## 1. プロジェクト概要
既存の「WalkingField」「PriceTracker」に機能追加と自動テストを実施し、さらに3つの新規アプリケーション「Todo & Pomodoro」「Markdown Memo」「Habit Tracker」を追加する統合プロジェクト。

## 2. アプリケーション別仕様

### 2.1 既存アプリ: WalkingField
* **既存機能**: 無限生成世界、スロット制インベントリ、地形改変、防衛設備、セーブ機能。
* **追加機能案**:
  - 新規バイオームの追加
  - サウンドエフェクトの追加（※可能な範囲で）
  - ミニマップの実装
* **テスト方針**: コアロジック（インベントリ管理、地形生成ロジックなど）の単体テスト。
* **備考**: PWA対応を想定。

### 2.2 既存アプリ: PriceTracker
* **既存機能**: 商品の価格推移をグラフで管理。
* **追加機能案**:
  - CSVエクスポート・インポート機能
  - 価格アラート設定（目標価格を下回ったかのチェック機能）
* **テスト方針**: 価格データ管理ロジック、グラフ用データ変換の単体テスト。
* **備考**: PWA対応を想定。

### 2.3 新規アプリ: Todo & Pomodoro
* **概要**: タスク管理とポモドーロタイマーを組み合わせたアプリ。
* **主な機能**:
  - タスクの追加、編集、削除機能
  - タスクの完了状態のトグル
  - 25分集中 / 5分休憩 のポモドーロタイマー
  - タイマーとタスクの連動（現在進行中のタスクの指定）

### 2.4 新規アプリ: Markdown Memo
* **概要**: Markdown形式で記述できるシンプルなメモアプリ。
* **主な機能**:
  - メモの作成、編集、削除
  - Markdownのリアルタイムプレビュー
  - ローカルストレージへの自動保存

### 2.5 新規アプリ: Habit Tracker
* **概要**: 毎日の習慣化したい項目をチェックするアプリ。
* **主な機能**:
  - 習慣項目の登録、削除
  - 日別の達成チェック（カレンダービューまたはリストビュー）
  - 継続日数の表示

### 2.6 新規アプリ: Expense Tracker (家計簿・支出管理ダッシュボード)
* **概要**: Next.jsの機能学習（サーバーコンポーネント、クライアントコンポーネント、カスタムフック、useEffect、propsの活用）を目的とした家計簿アプリ。
* **主な機能**:
  - サーバーコンポーネントでの初期モックデータ取得
  - 支出の追加・削除機能
  - カテゴリごとの支出集計
  - ローカルストレージを利用したデータの永続化
* **学習ポイント**:
  - コンポーネント間のデータの受け渡し（props）
  - 状態管理と副作用の扱い（useState, useEffect, カスタムフック）
  - サーバー/クライアントの境界の理解

## 3. 開発要件
* コードには、後から見返しても分かりやすいように冗長でも丁寧な日本語コメントを付与する。
* まとまった機能が完成した段階で、本仕様書を更新し、適切にGitコミットを行う。
