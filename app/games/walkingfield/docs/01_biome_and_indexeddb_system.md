# 開発記録 01: バイオーム多様化と IndexedDB チャンク保存システム

## 1. 湿度（Moisture）導入によるバイオームの多様化
初期の実装では「高さ（Height）」ノイズのみを使用して地形を決定していたため、地形が単調な層状（バウムクーヘン状）になっていました。これを改善するために「湿度」の概念を導入しました。

### 実装内容
- **デュアルノイズシステム**: `getTerrainHeight` に加え、座標オフセットをずらした `getTerrainMoisture` 関数を追加。
- **バイオームテーブルの導入**:
    - **乾燥地帯**: 砂漠 (🌵)、サバンナ (🏜️)、岩山 (⛰️)
    - **湿潤地帯**: ジャングル (🌴)、沼地 (🍄)、雪山 (🏔️)
    - **標準**: 草原 (🌱)、森 (🌲)
- **効果**: 同じ標高（平地や山）であっても、場所によって全く異なる景色が生成されるようになり、探索の楽しさが向上しました。

## 2. IndexedDB によるチャンクベースのセーブシステム
「マップの書き換え（建築）」を主軸にするため、従来の `localStorage` の制限を突破し、大規模建築に耐えうる基盤を構築しました。

### 背景と課題
- **容量制限**: `localStorage` は約5MBまでしか保存できず、大量の建築を行うとセーブ不能になるリスクがありました。
- **計算負荷**: 全タイルの変更を一つのリストで管理すると、データ増大時に検索処理が重くなり、ゲームのFPSが低下します。

### 解決策: チャンク管理への移行
- **IndexedDB の採用**: 数百MB〜GB単位のデータを保存可能なブラウザ内蔵DBに移行。
- **チャンク単位の保存**: 8x8マスの「チャンク」を一つの保存単位とし、`db.ts` を通じて非同期で読み書き。
- **動的ロード/アンロード**: プレイヤーの周囲のチャンクデータのみをメモリに読み込み、遠ざかったデータは解放することで、メモリ使用量を一定に保ちます。

### データの保存形式
- **`chunks` ストア**: `id: "cx,cy"`, `tiles: { "tx,ty": TileData }`
- **`player` ストア**: `id: "main"`, 座標、累計歩数など

## 3. 今後の課題・拡張
- **画像アセットへの差し替え**: 現在の絵文字ベースからタイルマップ画像への移行。
- **クラフト要素**: 特定のバイオームでしか手に入らない素材や、それらを使った建築物の追加。
- **マルチ保存**: 複数のセーブスロットへの対応。

## 4. 敵キャラクター（ゾンビ）と HP システムの導入
サバイバル要素として、プレイヤーに敵対するエンティティと体力の概念を実装しました。

### 実装内容
- **HP システム**:
    - プレイヤーに HP（最大100）を設定。
    - `GameOverlay` にリアルタイムで変動する体力ゲージを表示。
    - ダメージを受けた際に画面が赤くフラッシュする演出を追加。
- **敵 AI（ゾンビ）**:
    - プレイヤーの周囲にランダムにスポーン（🧟）。
    - プレイヤーを一定距離で感知し、追尾するシンプルな追跡 AI を搭載。
    - 接触時に 10 ダメージを与え、1秒間の無敵時間（連続ダメージ防止）を設定。
- **ゲームオーバー処理**:
    - HP が 0 になると、座標 `(0, 0)` にリスポーンし、HP が全回復します。
- **永続化**:
    - 現在の HP は `IndexedDB` に保存され、再開時に復元されます。

## 5. 昼夜サイクルと光源システム
時間の経過とともに世界の明るさが変化する「昼夜サイクル」と、暗闇を照らす「光源システム」を実装しました。

### 実装内容
- **昼夜サイクル**:
    - 現実時間の約2分間で1日が経過するサイクルを導入。
    - 昼 (☀) → 夕方 (🌅) → 夜 (🌙) → 明け方 (🌅) の順に画面の明るさが変化。
- **光源システム (Lighting)**:
    - 夜間は画面全体に暗闇のオーバーレイ（RenderTexture）を重ね、視界を制限。
    - プレイヤーの周囲、および設置された **松明 (🕯️)** や **焚き火 (🔥)** の周囲を円形に切り抜く（ブレンドモード: ERASE）ことで、光源による照光を表現。
- **新アイテム: 松明 (🕯️)**:
    - 宝箱から入手可能。
    - 地面に設置することで、夜間の安全な視界を確保できます。

## 6. テント休憩機能の強化とリスポーンシステム
「テント」を単なる保存場所から、戦略的な拠点としての機能を持つように拡張しました。

### 実装内容
- **休憩によるリセット効果**:
    - テントで宿泊（セーブ）した際、ゲーム内時間を強制的に「朝」にリセットします。
    - 宿泊時に周囲の敵（ゾンビ）をすべて消去し、安全を確保します。
- **動的リスポーン地点**:
    - 最後にテントで宿泊した場所を「リスポーン地点（復活場所）」として記憶します。
- **ゲームオーバーの改善**:
    - HP が 0 になった際、初期位置 `(0, 0)` ではなく、最後に休憩したテントの位置で復活するように変更しました。
    - 復活時も時間は「朝」にセットされ、体制を立て直しやすくなっています。

## 8. スロット制ホットバーとインベントリシステムの刷新
アイテム数の増加に対応し、サバイバル・クラフトゲームとしての操作性を向上させるため、インベントリシステムを全面的に刷新しました。

### 実装内容
- **スロット制ホットバー**:
    - 従来の無制限なアイテム一覧を廃止し、固定5枠のアイテムスロット（ホットバー）を導入。
    - アイテムの「所持数（スタック）」と「空きスロット」の概念を追加。
- **素手（Empty Hand）モード**:
    - 左端に固定の「素手（👊）」状態を追加。将来的な素材採取（木を叩くなど）の基盤となります。
- **アイテムの廃棄機能**:
    - 選択中のアイテムを「捨てる（🗑️）」機能を追加。不要なアイテムを整理してスロットを空けることが可能です。
- **自動アイテム取得**:
    - 宝箱に接触した際、空きスロットまたは同じアイテムのスロットに自動で補充される仕組みを構築。
- **永続化の更新**:
    - IndexedDB のデータ構造を `(InventorySlot | null)[]` に変更し、スロットの状態を完全に保存。

## 9. ピッケルによる地形改変システム
世界の地形をプレイヤーが直接作り変えることができる「整地」機能を実装しました。

### 実装内容
- **新アイテム: ピッケル (⛏️)**:
    - 宝箱から入手可能な耐久制の道具。
    - 使用すると地形の「高度」を一段階下げることができます。
- **チャネリング（作業時間）**:
    - 採掘には約1.5秒の作業時間が必要で、その間は移動できず無防備な状態になります。
    - プレイヤーの頭上にプログレスバーが表示されます。
    - 作業中に「移動しようとする」または「敵からダメージを受ける」と作業は強制キャンセルされます。
- **地形の段階的変化**:
    - 雪山 → 岩山 → 森/ジャングル → 草原 → 砂地 の順に地形が変化します。
    - これにより、険しい山を切り拓いて道を作ったり、拠点周囲を平地に整えたりすることが可能になりました。

## 10. 新アイテム：道路、バリケード、罠、街灯
拠点の構築と防衛、効率的な移動を可能にする新しい建築アイテムを追加しました。

### 追加アイテム
- **道路 (🛣️)**: 
    - 設置するとその上の移動速度が **1.5倍** になり、地形による移動ペナルティを完全に無視できます。
- **バリケード (🚧)**: 
    - プレイヤーもゾンビも通行できない強固な壁。拠点の囲いや侵入経路の制限に利用できます。
- **罠 / スパイク (🪤)**: 
    - 上に乗ったゾンビに継続ダメージを与えます。防衛ラインの構築に役立ちます。
- **街灯 (🏮)**: 
    - 松明や焚き火よりも広い範囲（半径約1.5倍）を照らします。設置場所は通行不可となります。
- **撤去機能**:
    - 全ての設置アイテムは「ピッケル」で採掘することで撤去し、元の自然な地形に戻すことができます。

## 11. スタック解消（緊急帰還）機能
プレイヤーが地形で身動きが取れなくなった際（スタック）の救済措置として、緊急帰還機能を実装しました。

### 実装内容
- **緊急帰還ボタン (🏳️)**:
    - 画面右上の「戻る」ボタンの横に、白旗（🏳️）のアイコンを追加しました。
    - このボタンを押すと確認ダイアログが表示され、承諾すると即座に「最後に休んだテント（または初期位置）」へテレポートします。
- **用途**:
    - バリケードで自分を囲んでしまった場合や、ピッケルを持たずに深い山や海に閉じ込められた場合など、ゲームの継続が困難な「詰み」の状態を解消するために利用できます。

## 7. 地形による移動速度の変化（移動コスト）
世界をよりリアルに感じ、探索の戦略性を高めるため、立っているバイオームに応じてプレイヤーの移動速度が変化する機能を実装しました。

### 実装内容
- **移動速度倍率の設定**:
    - **100% (通常)**: 草原 (🌱)、砂漠 (🌵)、サバンナ (🏜️)、砂浜 (🏖️)、橋 (🪵)
    - **80% (やや遅い)**: 森 (🌲)、ジャングル (🌴)、沼地 (🍄)
    - **60% (遅い)**: 岩山 (⛰️)
    - **40% (非常に遅い)**: 雪山 (🏔️)
- **効果**: 高い山を登る際や深い森を抜ける際の「困難さ」が表現され、目的地へ向かう際のルート選びに意味が生まれました。
